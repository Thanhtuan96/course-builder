Create two command files.

**commands/professor/capstone.md**
Frontmatter: name=professor:capstone, description="View your capstone project brief"
Body routing instructions:
- Read and display the contents of CAPSTONE.md for the active course.
- If sections are still in progress (not all Done): show the brief but remind the user they must complete all sections before starting the capstone. The capstone unlocks when the last section is marked done.
- If capstone is unlocked: show the brief and encourage them to start building.
- If no CAPSTONE.md exists: prompt them to run /professor:new-topic first.

**commands/professor/capstone-review.md**
Frontmatter: name=professor:capstone-review, description="Submit your finished capstone project for Socratic review"
Body routing instructions:
- First: Read COURSE.md and verify all course sections (non-capstone rows) have status Done.
  If any section is Not started or In progress, respond: "You need to complete all course sections before the capstone review. Check your progress with /professor:progress."
- When all sections are Done and user shares their project:
  1. Read the entire project carefully before commenting
  2. Overall assessment — one paragraph on what they built and whether it demonstrates the course objectives
  3. Section-by-section feedback — for each major part: what's working + Socratic question about a decision + one thing to research
  4. Standout moment — the single best thing they did
  5. One growth challenge — a concrete next step to level up the project independently
  6. Final verdict: "Course Complete" (update COURSE.md capstone status to Done with date) or "Almost There" (tell them exactly what's missing)
- Still no code writing during capstone review — feedback only.
- Include $ARGUMENTS.
