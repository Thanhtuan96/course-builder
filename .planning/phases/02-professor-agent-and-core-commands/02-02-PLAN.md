**commands/professor/new-topic.md**
Frontmatter: name=professor:new-topic, description="Start a new Socratic learning course on a topic", argument-hint="[topic]"
Body routing instructions:
- Use AskUserQuestion with exactly these 4 questions in a single call:
  1. What do you want to learn?
  2. What is your current experience with this topic?
  3. What level are you aiming for? (Beginner / Intermediate / Advanced / Expert)
  4. What do you want to be able to build or do after this course?
- Propose the syllabus inline in chat. Do NOT write COURSE.md yet.
- Frame each section description around how it moves the user toward their stated goal.
- After the user confirms (conversationally), write COURSE.md and CAPSTONE.md simultaneously.
- Include $ARGUMENTS at the end (user may pass topic directly).

**commands/professor/next.md**
Frontmatter: name=professor:next, description="Load the next section of your course"
Body routing instructions:
- Load the next not-started section from COURSE.md into LECTURE.md.
- Update COURSE.md status to In progress for that section.
- Present the section content in chat and prompt the user to attempt the exercise.

**commands/professor/done.md**
Frontmatter: name=professor:done, description="Mark the current section complete after demonstrating understanding"
Body routing instructions:
- Ask the user to explain the core concept of the current section in their own words before marking done.
- If explanation is solid: mark section Done in COURSE.md, add progress log entry, check if all sections are done and trigger capstone unlock if so.
- If explanation is shaky: give Socratic feedback, do not mark done until understanding is demonstrated.

  <action>
Create three thin command files.

**commands/professor/review.md**
Frontmatter: name=professor:review, description="Review your work with structured Socratic feedback"
Body routing instructions:
- Use the 4-step Socratic review structure:
  1. What's working — acknowledge what they got right (1-2 sentences)
  2. Socratic question — point to the most important gap with a question, not a correction
  3. One concept to study — name it, do not explain it fully
  4. Next action — one clear thing to try
- Focus on one issue at a time. Do NOT rewrite their code.
- Include $ARGUMENTS (user may share code/answer inline).

**commands/professor/syllabus.md**
Frontmatter: name=professor:syllabus, description="View the full course outline and progress"
Body routing instructions:
- Read and display the contents of COURSE.md for the active course.
- If no active course exists, prompt the user to run /professor:new-topic.

**commands/professor/progress.md**
Frontmatter: name=professor:progress, description="View your learning progress and weak areas"
Body routing instructions:
- Read COURSE.md and produce a structured summary including:
  - Sections completed (with dates)
  - Current section and its status
  - Sections remaining
  - Concepts demonstrated well (from session review history)
  - Concepts that seemed shaky
  - Estimated time to finish
  - Capstone project status
