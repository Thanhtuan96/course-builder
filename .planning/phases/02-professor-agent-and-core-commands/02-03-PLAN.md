**commands/professor/hint.md**
Frontmatter: name=professor:hint, description="Get the next hint layer without revealing the answer"
Body routing instructions:
- Give the next hint layer for the current exercise.
- Infer the correct layer from the conversation history (how many professor:hint calls this section).
- Do not skip layers. If the user requests a 4th hint, suggest professor:stuck instead.
- Include $ARGUMENTS (user may describe what they're stuck on).
Note: Keep the body short — layer definitions and rules are in agents/professor.md.

**commands/professor/stuck.md**
Frontmatter: name=professor:stuck, description="Get structured guidance when genuinely blocked"
Body routing instructions:
- Before diving into the breakdown, use AskUserQuestion to ask: "Walk me through what you've tried so far."
  Offer structured options: "I described it below" / "Nothing yet" / "Tried X but got stuck on Y"
- Based on their response: identify the exact sticking point, break the problem into smaller steps,
  give a worked analogy using a different simpler example, guide to the next smallest step.
- Do NOT write the solution.
- Include $ARGUMENTS.

Create two thin command files.

**commands/professor/discuss.md**
Frontmatter: name=professor:discuss, description="Discuss a concept — conceptual Q&A only"
Body routing instructions:
- Open free-form discussion on the current topic or the user's question.
- Answer conceptually — no full code dumps.
- If the user asks "how do I do X?", respond with "What have you tried so far?" before any guidance.
- Use analogies, real-world comparisons, and ASCII diagrams where helpful.
- Keep responses conversational, not lecture-style.
- Include $ARGUMENTS (user passes the concept or question).

**commands/professor/quiz.md**
Frontmatter: name=professor:quiz, description="Run a quiz on your current topic", argument-hint="[section or concept]"
Body routing instructions:
- Use AskUserQuestion to ask the scope before generating:
  Options: "Current section" / "Entire course so far" / "Specific concept (describe below)"
- Generate 5 questions matched to the user's level and selected scope.
- After the user answers, review each answer with Socratic feedback — never just say "correct" or "wrong".
- Include $ARGUMENTS.
